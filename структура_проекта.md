Пояснение ключевых решений
Элемент	Почему так
app/core/	Всё, что относится к инфраструктуре (БД, конфиг, исключения)
app/models/	ORM-модели (каждая в своём файле)
app/schemas/	Pydantic-схемы (каждая в своём файле)
app/services/	Бизнес-логика (каждый сервис отвечает за свою сущность)
app/api/v1/	Роутеры (версионирование API на будущее)
app/utils/	Утилиты без состояния (чистые функции)
migrations/	Без Alembic (ручные SQL-скрипты)
scripts/	Вспомогательные скрипты для разработки
tests/	unit + integration (разделены по назначению)
Файлы, которые точно не разрастутся
Файл	Предполагаемый размер
main.py	~50 строк
config.py	~30 строк
database.py	~40 строк
exceptions.py	~20 строк
dependencies.py	~20 строк
generate.py (роутер)	~150 строк
Итого	~310 строк
Файлы с потенциальным ростом
Файл	Предполагаемый размер	Что делать при росте
generate_service.py	~150-200 строк	Разбить на context_builder.py, response_handler.py
sync_service.py	~100-150 строк	Вынести disk_scanner.py и db_updater.py
snapshot_service.py	~100-150 строк	Пока норм
file_service.py	~80-100 строк	Пока норм
Критерии для разделения
Файл > 300 строк → вынести часть логики в отдельный модуль

Файл > 500 строк → обязательно разделить

Класс > 200 строк → вынести методы в отдельные сервисы
